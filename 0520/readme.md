# 0520 과제
웹프응

안녕하십니까 스프링부트, 조건부 N빵 정산 플랫폼에 대한 발표를 시작하겠습니다
(넘기기)
먼저 이번 프로젝트의 핵심 기반인 스프링부트에 대해 간단히 알아보고,
스프링과 안티그래비티의 연계과정을 소개해 드리겠습니다.
이어서 팀원들이 어떻게 소통했는지 협업 과정과
백엔드 및 프론트의 연결 과정을 말씀드리고
플랫폼 시연을 끝으로 발표를 마무리 하도록 하겠습니다.
(넘기기)
그럼 첫 번째 순서로 이번 프로젝트의 뼈대라고 할 수 있는 스프링 부트에 대해 알아보겠습니다.
(넘기기)
흔히들 스프링 과 스프링 부트 를 혼용해서 부르곤 하는데요.
스프링 부트는 기존 스프링의 복잡함을 해결하기 위해 나왔습니다.
원래 스프링은 많은 기업이 사용할 만큼 강력하고 안정적인 프레임워크입니다.
하지만 설정이 너무 복잡해서 실제 개발을 시작하기도 전에 많은 시간을 쏟게됩니다.
이 문제를 해결한 것이 바로 스프링 부트 입니다.
스프링의 강력함은 가져가되, 초보자도 쉽게 접근할 수 있도록 진입 장벽을 확 낮췄습니다.
덕분에 개발자들이 복잡한 환경 설정에 시간을 쏟는 대신,
서비스 기능을 만드는 데만 집중할 수 있게 되었습니다.
(넘기기)
그렇다면 스프링 부트가 구체적으로 어떤 편리함을 주는지 알아보겠습니다.
첫번째는 내장 서버 입니다.
예전에는 웹 서비스를 구동하려면 별도로 서버를 설치하고 세팅해야 했지만,
이제는 프로그램 안에 서버가 이미 내장되어 있어
그냥 실행 버튼 하나만 누르면 바로 웹 서비스가 돌아갑니다.
두번째는 자동 구성 입니다.
말 그대로 스프링부트가 개발자의 의도를 파악해
알아서 최적의 세팅을 잡아주는 기능입니다.
세번째는 스타터 의존성 입니다.
필요한 기능 묶음을 패키지 형태로 제공해주기 때문에,
라이브러리 간의 버전 충돌을 걱정할 필요가 없습니다.
마지막으로 운영 기능 까지 지원합니다.
이 모든 특징을 요약하자면,
개발자가 오직 서비스의 가치를 만드는 것에만 신경 쓰게 해준다는 것 입니다.
(넘기기)
과거 개발자들을 괴롭히던 설정 지옥 에서 해방되었고, 귀찮은 서버 세팅을 생략할 수 있으며, 라이브러리 간의 버전 충돌 스트레스가 사라졌습니다. 
그리고 전 세계적으로 사용자가 많고 탄탄한 커뮤니티가 형성되어있습니다.
(넘기기)
스프링부트 덕분에 과거보다 편해진 것은 사실입니다.
하지만 여전히 프로젝트를 처음 시작할 때 환경을 세팅하는 과정은 번거롭고 까다롭습니다.
또한 비슷한 구조의 파일을 매번 똑같이 만들어야 하는 반복 코드 문제도 있습니다.
(넘기기)
Antigravity는 자율형 AI 에이전트를 통해 초기 세팅의 불편함을 최소화하고,
보일러 플레이트를 자동화 합니다.
그래서 스프링부트와 안티그래비티가 만나면 개발 속도가 비약적으로 향상합니다.
또 AI가 제안하는 기능들을 실제 서비스로 즉시 연결 할 수 있고,
프론트엔드와 백엔드의 연동이 쉬워집니다.
(넘기기)
다음은 저희 팀의 협업과정에 대해 설명드리겠습니다.
(넘기기)
이번 프로젝트는 단순히 개발만 진행한 것이 아니라, 
기획부터 개발, 배포까지 역할을 체계적으로 분리하여 진행하였습니다.
먼저 아이디어 회의 단계에서는 카카오톡과 FIgma를 활용하였습니다.
카카오톡을 통해 팀원간 의견 공유 및 피드백을 진행 하였고,
Figma에서 아이디어를 정리하였습니다.
기획 단계에서는 Notion과 gemini를 활용하여 PRD를 작성하고
개발 단계에서는 안티그래비티와 스프링부트를 함께 사용하였습니다.
안티그래비티를 활용해 프론트엔드와 일부 기능 구현을 빠르게 진행하고,
스프링부트를 통해 백엔드 구조와 API 서버를 구축하였습니다.
API는 Gemnin API와 Google Vision API를 활용하였습니다.
디비 는 수파베이스를 사용하여 실시간 데이터 관리와 간편한 백엔드 연동을 가능하게했고,
소스관리는 깃허브를 기반으로 백엔드, 프론트엔드, PRD 문서 등을 깃허브에서 통합 관리 하였습니다.
배포 단계에서는 Vercel과 엔그록을 활용하였습니다.
(넘기기)
저희가 선택한 아이디어는 스냅스플릿, 스마트 영수증 정산 플랫폼 입니다.
영수증 촬영 한번으로 항목별 텍스트를 추출하고 조건부 분할 계산부터
카카오톡 송금 요청과 토스 연동까지 해결하는 정산 플랫폼 입니다.
(넘기기)
PRD는 팀원들끼리 공유하는 PRD와 Antigravity용 PRD를 나누고,
업그레이드할때 버전별로 정리하여 프로젝트를 진행해
개발 의도 전달의 정확성을 높였습니다.
(넘기기)
저희는 깃허브에서 메인 브랜치를 기준으로 프론트엔드 백엔드 개발 영역을 나누어 작업하였고, main 브랜치에 머지하여 통합하였습니다.
