# react를 사용하는 이유

- 1. 컴포넌트(Component) 기반 개발
     React의 가장 큰 특징은 UI를 **재사용 가능한 독립적인 단위(Component)**로 쪼개서 관리한다는 점입니다.

     효율성: 한 번 만든 버튼, 입력창, 네비게이션 바를 여러 페이지에서 그대로 가져다 쓸 수 있습니다.
     유지보수: 특정 부분에 문제가 생기면 해당 컴포넌트 코드만 수정하면 되므로 관리가 매우 쉽습니다.

- 2. 가상 DOM (Virtual DOM)을 통한 성능 최적화
     전통적인 방식은 데이터가 바뀔 때마다 브라우저의 DOM 전체를 다시 그립니다. 이는 규모가 큰 서비스에서 성능 저하의 주범이 됩니다.

     작동 원리: React는 실제 DOM을 직접 건드리기 전에 메모리에 가상 DOM을 먼저 만듭니다.
     차이점 업데이트: 변경사항이 생기면 가상 DOM끼리 비교하여 실제로 바뀐 부분만 골라 실제 DOM에 적용(Diffing & Patching)합니다. 결과적으로 브라우저의 연산 부담이 확 줄어듭니다.

- 3. 선언적 UI (Declarative)
     React는 UI가 어떻게 보일지를 미리 설계하는 "선언적" 방식을 사용합니다.

     명령형(기존): "버튼을 찾아서, 클릭 이벤트를 달고, 텍스트를 A에서 B로 바꿔라"라고 일일이 지시해야 합니다.
     선언적(React): "데이터가 A이면 이 화면을 보여주고, B이면 저 화면을 보여줘"라고 상태($state$)에 따른 결과값만 정의합니다. 코드가 훨씬 직관적이고 예측 가능해집니다.

- 4. 강력한 생태계와 커뮤니티
     이게 어쩌면 실무에서 가장 큰 이유일지도 모릅니다.

     방대한 라이브러리: 상태 관리(Redux, Recoil), 라우팅(React Router), 스타일링(Styled-components) 등 이미 검증된 도구들이 널려 있습니다.
     인력 풀: 많은 개발자가 사용하므로 팀 빌딩이 쉽고, 문제에 부딪혔을 때 구글이나 Stack Overflow에서 해결책을 찾기 매우 쉽습니다.

- 5. 단방향 데이터 흐름 (One-way Data Binding)
     데이터가 부모에서 자식으로만 흐르는 규칙을 가집니다.데이터의 변화를 추적하기 쉽고, 디버깅이 명확해집니다. 앱이 커져도 데이터가 어디서 왔는지 꼬이는 일이 적습니다.
